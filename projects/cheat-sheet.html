Functions

Keeping my head straight around how to use functions as well as flow in the code is a bit of task, as such I figured I would reiterate how to declare functions in both Javascript and Ruby. Well, mainly in Javascript.

Ruby

Defining a method in Ruby

def a_method
	# do something
	return something
end

yayz = a_method

Boom. Done. And the last line calls the method and assigns it to a variable.

Javascript

Defining a function in JS, as a function declaration

function aDeclaredFunction {
	//do something
	return something;
}

var yayz = aDeclaredFunction;

Okay...easy, right? Hold tight...

var aFunctionExpression = function { 
	//do something
	return something;
 }
var yayz = aFunctionExpression

Say what? Well, it's called a function expression. It allows you to assign an anonymous function to a variable, and that function gets called whenever that variable is used. (You can also name a function expression, but that's less common.)

And this is where the fun part starts. If you use a function declaration (first example), it means that the function is moved to the top of the metaphysical code flow. As such, you can call a function before you declare it. 

var yayz = aDeclaredFunction;

function aDeclaredFunction {
	//do something
	return something;
}

And the last way to make a function in JS is to use a function constructor. Since functions are objects in JS, they have a constructor. 

var aConstructedFunction = new Function() {
	// do something
	return something
}

var yayz = aConstructedFunction;

I said that this example was the last way to declare a function in JS...well, there is one more way, except things get all kind of weird because you can both express the function and the immediately invoke it. 

(function() {
	// do something
	return something
})();

Woah.

Some fun facts about functions for you...

Ruby

  • Ruby does NOT have nested methods. (More explanation)
  • Functions are normally referred to as methods, period. (Well, actually, most argue that there aren't even functions in Ruby, as functions are objectless methods, and Ruby is a completely OO language).
  • Ruby has closures! But they are weird, too
  • Ruby has these interesting things called Procs and lambdas that can help you create a self invoking block of code.

Javascript

  • Javascript totally has nested methods, yippee!
  • A method is a function inside of an object.
  • Variable scoping is super important in Javascript, and self invoking functions have a fun thing called closures.
