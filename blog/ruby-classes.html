Class It Up

Ruby's Classes

11/30/15

When I first learned about classes, it was a heady concept to wrap my head around. Classes create objects that have specific methods and variables? Huh? (Let alone the fact that I didn't really grasp what a method or a variable was in the first place. Or the fact that a method is really just a class-based function.)

But I digress! Classes and objects. A class is like a template, and objects are the actual occurence/creation of that template. For example, let's say I wanted to make an RPG crime noir text based Ruby console game. (Inspiration from Learn Ruby the Hard Way, Exercise 43.) And let's say I wanted to have a character, a detective. As such, I could write it like this: 

      class Detective 
        def initialize(name)
          @@weapon = ["knife", "gun", "cigarettes"]
          @name = name
        end

        def tell_me_about_yourself
          puts "My name is Detective #{name}, and I fight the crime lords and underlings of this rotten city."
          puts "I have a #{print_weapon_list}."
        end

        def add_weapon(weapon)
          @@weapon.push(weapon)
        end

       def print_weapon_list
          count = 0
          while count < @@weapon.length - 2
            print @@weapon[0] + ", "
            count += 1
          end
          print "and" + @@weapon[@weapon.length-1]
        end
      end

      a_detective = Detective.new("Detective Bronson")

So essentially this code is creating a new detective object. You can pass in a name for your detective character through the initialize method. The initialize method creates a set of instance variables (variables which can be used across all instances of this particular class, as opposed to a local variable which can be used in a particular method).

The detective class comes with a few other fun tricks - you can add weapons to the detective's arsenal through the add_weapon methods, and you can get some info on your detective, through the tell_me_about_yourself method. That particular method calls another method to help it out with printing the list of weapons the detective currently owns.

All in all, classes are fun because they all for more interesting relationships between data and methods. Meanwhile, I will happily continue sleuthing out better ways to use them!
